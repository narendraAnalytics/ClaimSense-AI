# backend/ — CLAUDE.md

Guidance for Claude Code when working in this directory. See the repo-root `CLAUDE.md` for overall project context.

## Stack
- Python ≥3.12, managed with `uv` (`pyproject.toml` + `uv.lock`, venv at `.venv/`)
- FastAPI (`fastapi[standard]`) for the HTTP API
- LangGraph + LangChain for multi-agent orchestration
- Pydantic / Pydantic Settings for schemas and config
- PyMuPDF + Pillow for document/image handling
- fpdf2 for PDF report generation (Phase 9, pure-Python, no system libraries needed)
- Sarvam AI (Sarvam-30B reasoning, Sarvam Vision OCR) as the LLM provider — OpenAI-compatible API, called via `httpx`/LangChain, not a dedicated SDK
- Qdrant Cloud (`qdrant-client[fastembed]`) for Historical Similarity's semantic vector search (Phase 10) — embeddings generated locally via FastEmbed (`BAAI/bge-small-en`, ONNX, no GPU/torch, no extra API key), not Sarvam (which has no embeddings endpoint)
- loguru for logging
- pytest (dev dependency, `[dependency-groups] dev` in `pyproject.toml`) for tests — run with `uv run pytest`

Run commands: `uv sync` to install deps, `uv run uvicorn app.main:app --reload` to run the dev server (`/api/v1/health`, `/docs`, `/redoc` all live at that point).

**Deployed**: live on Railway at `https://claimsense-ai-production.up.railway.app` (`APP_ENV=production`), e.g. `GET https://claimsense-ai-production.up.railway.app/api/v1/health`.

  **Two real bugs found and fixed during live verification**: (1) `fpdf2`'s `multi_cell()` defaults `new_x` to `RIGHT`, not `LMARGIN` — after the first full-width `multi_cell` call the cursor lands at the right margin, so the *next* `multi_cell(0, ...)` call computes near-zero available width and raises `FPDFException: Not enough horizontal space to render a single character`. Fixed by passing `new_x=XPos.LMARGIN, new_y=YPos.NEXT` to every `multi_cell` call. (2) the core `helvetica` font only supports Latin-1, and LLM-generated reasoning text can contain smart quotes/em dashes/currency symbols that would otherwise crash mid-report — fixed with a `_sanitize()` helper that encodes to `latin-1` with `errors="replace"` before rendering.

Fixed two ways: (1) `app/prompts/fraud.md` was rewritten to make the narrative-vs-diagnosis comparison mechanical and explicit, to state plainly that Medical's/Billing's "validated" labels are document-internal checks only and don't bless the claim's overall story, and to require a fraud_score floor of 50 whenever `narrative_medical_consistency` is false; (2) `app/services/fraud.py::convert_to_model` now enforces that floor **in Python** (`_NARRATIVE_MISMATCH_FRAUD_FLOOR = 50`) — forcing `fraud_score` up and ensuring a narrative-mismatch entry in `red_flags` whenever the model itself reports `narrative_medical_consistency: false` — the same "don't trust the model's own arithmetic/self-report for anything consequential" pattern already used for `payable_amount` and `fraud_level` elsewhere in this codebase. `test_phase9_workflow.py` was updated to assert this explicitly (`narrative_medical_consistency is False`, `fraud_score >= 50`, a narrative red flag present, `settlement.approval_status == "need_review"` instead of `"approve"`). **Not yet re-run against the live API** — Sarvam credits were exhausted by this session's extensive live testing; re-run `test_phase9_workflow.py` once credits are topped up to confirm.

  - **Parallelize Billing/Fraud/History** — `app/graph/builder.py` rewired from the linear `medical -> billing -> fraud -> history -> settlement` chain to a fan-out/fan-in: `medical -> {billing, fraud, history}` then `{billing, fraud, history} -> settlement` (LangGraph's native multi-edge pattern, no `Send` API needed since the fan-out is static). All three write disjoint `ClaimState` keys, but all three also independently stamp `updated_at: datetime.now(...)` — a plain non-reducer field — which raises `InvalidUpdateError` when three nodes write it in the same superstep. Fixed with a `_latest()` reducer on `updated_at` in `app/graph/state.py` (a thin wrapper function, not the builtin `max` directly — LangGraph's reducer detection calls `inspect.signature()` on it, which raises on `max` since it has none). Accepted trade-off: Fraud/History both read `state.get("billing_result")` for extra context; running in the same superstep as Billing means they now see `None` instead of Billing's finished output — both call sites already degraded gracefully on `None`, and Settlement (after the join) still sees the real value, so this is a soft quality trade-off, not a bug. `test_claims_workflow.py`/`test_medical_agent.py`/`test_phase9_workflow.py`'s `workflow_history` assertions changed from an exact ordered list to prefix/parallel-set/suffix checks, since the three parallel stages' relative order isn't guaranteed.

Only completed-claim summaries are embedded — never raw documents. Uses Qdrant's built-in FastEmbed integration (`BAAI/bge-small-en`, local ONNX, no GPU/torch, no extra API key — Sarvam has no embeddings endpoint, confirmed via docs search) and the current non-deprecated `upsert()`/`query_points()` API with explicit `models.Document` wrapping (not the convenience `add()`/`query()` helpers, which still work but emit a deprecation warning — confirmed the non-deprecated pattern via web search after that warning surfaced mid-build). ## Conventions
- Agent folders stay self-contained: `__init__.py`, `agent.py`, `prompt.py` — an agent's logic and prompt live together, not scattered.
- Keep prompts in `app/prompts/*.md`, not inlined in Python — they should be reviewable/versionable independently of code.
- Routes in `app/api/v1/` should stay thin; put logic in `app/services/` or `app/agents/`, not in route handlers.
- Env vars currently defined (see `.env.example`, real values in `.env` which is gitignored):
  - `SARVAM_API_KEY` — Sarvam AI (reasoning + OCR; same key used by both `SarvamVisionClient` and `SarvamChatClient`, no separate chat key)
  - `CONVEX_URL`, `CONVEX_DEPLOY_KEY` — Convex DB/backend
  - `QDRANT_API_KEY`, `QDRANT_URL` — Qdrant Cloud vector search; **wired up as of Phase 10** (`app/services/qdrant_client.py`/`history_store.py`/`history.py`) — no longer just present-but-unused
